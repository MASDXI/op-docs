# Building Apps on OP Stack

Welcome to the world of building decentralized applications on the OP Stack! Whether you're building on OP Mainnet or other OP Stack chains, this guide will help you get started.

## Getting Started

Building on OP Stack chains is nearly identical to building on Ethereum. The OP Stack is EVM-compatible, which means you can use all your favorite Ethereum development tools and libraries.

## Key Differences from Ethereum

While OP Stack chains are EVM-compatible, there are some differences to be aware of:

<Accordion title="Transaction Fees">
Transaction fees on OP Stack chains consist of two components:
- **L2 Execution Fee**: Cost of executing the transaction on L2
- **L1 Data Fee**: Cost of posting transaction data to L1
</Accordion>

<Accordion title="Block Time">
OP Stack chains typically have faster block times than Ethereum:
- **OP Mainnet**: ~2 seconds
- **Ethereum**: ~12 seconds
</Accordion>

<Accordion title="Finality">
Transaction finality works differently:
- **Unsafe**: Immediate but can be reorganized
- **Safe**: Submitted to L1 but not yet finalized
- **Finalized**: Confirmed on L1 and immutable
</Accordion>

## Best Practices

### Gas Optimization
- Use efficient smart contract patterns
- Minimize data stored on-chain
- Batch operations when possible
- Consider L1 data costs for large transactions

### Security
- Audit your smart contracts
- Use established libraries like OpenZeppelin
- Test thoroughly on testnets
- Consider the implications of cross-chain interactions

### User Experience
- Implement proper error handling
- Provide clear transaction status updates
- Consider transaction finality levels
- Optimize for mobile users
