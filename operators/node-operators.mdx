# Node Operators

Node operators run the infrastructure that powers OP Stack networks. This includes running rollup nodes, execution clients, and other supporting services.

## What is a Node Operator?

A node operator runs the infrastructure that keeps OP Stack networks operational. This includes:

- Running rollup nodes (op-node)
- Operating execution clients (op-geth)
- Maintaining OP Supervisor services
- Providing RPC endpoints
- Ensuring high availability

## Node Architecture

### Core Components

<Accordion title="Rollup Node (op-node)">
The rollup node is responsible for:
- Deriving L2 blocks from L1 data
- Participating in the consensus protocol
- Validating block proposals
- Handling peer-to-peer networking
</Accordion>

<Accordion title="Execution Client (op-geth)">
The execution client handles:
- Processing transactions
- Maintaining state
- Providing RPC endpoints
- Executing smart contracts
</Accordion>

<Accordion title="OP Supervisor">
The OP Supervisor enables:
- Cross-chain message validation
- Interoperability between chains
- Safety level determination
- Log event tracking
</Accordion>

## Hardware Requirements

### Minimum Requirements
- **CPU**: 4 cores
- **RAM**: 8 GB
- **Storage**: 100 GB SSD
- **Network**: 10 Mbps

### Recommended Requirements
- **CPU**: 8+ cores
- **RAM**: 16+ GB
- **Storage**: 500+ GB NVMe SSD
- **Network**: 100+ Mbps

## Configuration

### Basic Configuration
```yaml
# op-node configuration
l1-endpoint: "https://ethereum-rpc-endpoint"
l2-endpoint: "http://op-geth:8545"
rollup-config: "/path/to/rollup.json"
rpc-addr: "0.0.0.0"
rpc-port: 8547
```

### Advanced Configuration
- **P2P Networking**: Configure peer discovery and networking
- **Metrics**: Enable monitoring and metrics collection
- **Logging**: Set up comprehensive logging
- **Security**: Configure firewall and access controls
